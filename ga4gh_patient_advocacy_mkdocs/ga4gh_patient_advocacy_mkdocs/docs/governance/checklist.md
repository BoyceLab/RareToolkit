# Implementation checklist

## Before launch

- [ ] Define registry goals and priority use cases.
- [ ] Draft core intake questionnaire.
- [ ] Draft annual update questionnaire.
- [ ] Identify diagnosis-specific modules.
- [ ] Build a data dictionary.
- [ ] Map core phenotype items to HPO.
- [ ] Decide how disease labels and test types will be standardized.
- [ ] Set consent and recontact policies.
- [ ] Decide how report upload will work.
- [ ] Test the survey with families and revise for burden.

## After launch

- [ ] Monitor completion and drop-off.
- [ ] Review confusing items every quarter.
- [ ] Add new structured items for frequent free-text concepts.
- [ ] Maintain mapping version history.
- [ ] Publish community updates on what the registry is enabling.
- [ ] Prepare a minimal Phenopacket export for partner sharing.
